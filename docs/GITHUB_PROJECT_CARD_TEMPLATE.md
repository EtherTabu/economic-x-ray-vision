# GitHub Project Card Template

Use these compact cards in profile READMEs, portfolio docs, or repository index sections. Replace placeholders before publishing.

## Markdown Card

```markdown
### <Project Name>

<One-sentence truthful description.>

- Status: <active / prototype / research / archived>
- Stack: <TypeScript, Next.js, SQLite, etc.>
- Signal: <what makes this project interesting or credible>
- Repo: <REPO_URL>
```

## Table Row

```markdown
| <Project Name> | <What it does in one phrase> | <Stack> | <Status> |
```

## Featured Portfolio Card

```markdown
### <Project Name>

**Mission:** <Specific mission.>

**Why it matters:** <One sentence explaining the real-world or technical relevance.>

**Built so far:** <3-5 concrete shipped capabilities.>

**Stack:** <Stack.>

**Status:** <Current truthful state.>

**Repo:** <REPO_URL>
```

## Adaptation Rules

- Keep project descriptions short and factual.
- Do not claim users, revenue, production deployments, sponsors, or benchmarks unless they are real.
- If a project is early, say prototype or research.
- If a link is unknown, use `<REPO_URL>` instead of inventing one.
