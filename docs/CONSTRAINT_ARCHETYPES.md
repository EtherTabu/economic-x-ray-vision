# Constraint Archetypes

Constraint archetypes are recurring bottleneck patterns that appear across different systems and industries. They let the project say: this looks like the same type of constraint as another record, but in a different domain.

## Why Archetypes Matter

Industries often describe their problems with sector-specific language, but the underlying pattern can be familiar:

- A queue waits for review.
- A document package keeps bouncing back.
- A handoff loses ownership.
- A scarce tool or part controls the schedule.
- A support channel absorbs unresolved upstream friction.

Archetypes make those shared patterns explicit.

## Cross-Industry Analog Examples

- Queue backlog can appear in healthcare prior authorization, grid interconnection, permit review, public grant administration, and public records requests.
- Documentation chase can appear in medical necessity review, transmission permitting, mining review, occupational licensing, and compliance reporting.
- Equipment lead time can appear in grid transformers, semiconductor tools, replacement parts, and heavy industrial operations.
- Vendor qualification can appear in healthcare credentialing, contractor qualification, procurement onboarding, and critical material qualification.

## How Analogs Work

The analog engine compares records across different industries using shared archetypes, affected systems, intervention hints, categories, opportunity types, and score proximity. It does not claim that two industries are identical. It identifies reusable bottleneck patterns that may suggest what to measure or test next.
